# 多關卡保證金申退與退費流程 (Refund Workflow) 實作計畫

此計畫旨在為雛形系統加入完整的**退費與保證金退還機制**，讓民眾可以在前台申請退款（上傳存摺封面），並且讓後台的行政、會計、出納能依序簽核退撥。

## User Review Required

> [!IMPORTANT]
> 為了讓這套系統更貼近您的實際業務，退費除了原本的「承辦」與「出納」外，是否需要再增加一個專屬的「會計/場長 (Accounting/Director)」關卡來做模擬展示？目前計畫會先拆成：**承辦初審 $\rightarrow$ 會計核定 $\rightarrow$ 出納撥款** 三個步驟。若確認此流程符合現況，請點擊同意即可開始實作！

## Proposed Changes

### 資料結構與 Mock Data 更新

將在系統底層引入退費狀態機，分離「訂單狀態」與「退費狀態」，使已結案或已取消的訂單也能走退費流程。

#### [MODIFY] [bookings.ts](file:///Users/sungjunghung/Projects/miaoliSpaceRent/prottype/src/stores/bookings.ts)
- 新增 `RefundStatus` 型別定義：
  - `none`：無需退費
  - `admin_review`：民眾已送出申請，等待承辦人查核場地復原照與按鈕。
  - `accounting_review`：承辦送交會計/場長核定退費金額。
  - `cashier_processing`：會計核准，出納準備開立實體支票或匯款。
  - `completed`：退款完成。
- 在 `Booking` 介面新增：`refundStatus`、`refundAmount`、`refundBankbookImage`、`refundReason`。

#### [MODIFY] [bookings.json](file:///Users/sungjunghung/Projects/miaoliSpaceRent/prottype/src/mocks/bookings.json)
- 修改 `id: 62` 等特定幾筆已完成/已取消的 mock 資料，注入退款狀態，方便我們開發完一打開畫面就有現成案例可以 Demo。

### 民眾前台 UI (Portal)

民眾可以在會員中心看見退款進度，並且主動上傳自己的存摺影本。

#### [MODIFY] [BookingDetail.vue](file:///Users/sungjunghung/Projects/miaoliSpaceRent/prottype/src/view/portal/member/BookingDetail.vue)
- **申請退費 Modal**：當訂單狀態為 `cancelled` 時，顯示取消退費流程資訊。點擊後會彈出視窗請民眾填寫退款理由並上傳「銀行存摺封面」。
- **退費進度條**：套用既有的進度條 Component 樣式，當 `refundStatus !== 'none'` 時，在畫面上方顯示「退費審查進度（承辦 $\rightarrow$ 會計 $\rightarrow$ 出納）」。

### 管理員後台 UI (Admin)

加入多權限切換面板，讓不同處室的長官可以點自己的按鈕。

#### [MODIFY] [detail.vue](file:///Users/sungjunghung/Projects/miaoliSpaceRent/prottype/src/view/admin/bookings/detail.vue)
- 在我們先前寫好的「權限切換區塊」中，新增 `accounting`（會計）的角色。
- 新增「退款處理區塊」：
  - 顯示民眾上傳的存摺圖片與金額。
  - 根據當下的角色顯示對應的核准按鈕：
    - 若切換為 **[承辦]**：只能操作 `admin_review` $\rightarrow$ 確認場地無損壞，交給會計。
    - 若切換為 **[會計]**：只能操作 `accounting_review` $\rightarrow$ 金額查核，准予退費。
    - 若切換為 **[出納]**：只能操作 `cashier_processing` $\rightarrow$ 確認匯出/開票完成。
